Задача:

С помощью паттерна проектирования Proxy, создать приложение, которое будет обращаться к ресурсу Chief и получать данные о статусе заказа и список заказов.
Обращение к стороннему ресурсу закешировать и залогировать.

Решение:
- 1. Создаем имитацию стороннего ресурса Chief, к которому клиент будет обращаться за статусами и заказами. Он будет содержать два метода которые вернут Dictionary статусов и List заказов.
- 2. Создаем метод GetStatuses(), который вернет нам статусы - готово, не готово, готовится. Имитируем задержку получения статусов - 2 секунды.
- 3. Создаем класс заказов, в нем свойства Id, Имя и Статус Id
- 4. Создаем метод GetOrder, который будет возвращать заказы и добавляем три заказа.
- 5. Создаем интерфейс IChief, который содержит два метода Chief.
- 6. Создаем класс ProxyChief и реализуем интерфейс IChief. ProxyChief содержит ссылку на ресурс Chief, Dictionary со статусами, которые мы единожды запросим у Chief, конструктор.
- 7. В Program создаем экземпляр класса ProxyChief и в качества параметра конструктора передаем экземпляр класса Chief.
